# 同济大学SITP项目 - 自研路面病害识别系统实验（操作）指南

本指南意在指导团队成员及评委，如何在配有现代显卡（如 RTX 5060）的本机系统中，从零开始走通从**数据集导入、模型训练、参数评估出图，到最终将其部署至 Web UI 平台**的全部工程实践流。

---

## 步骤一：数据准备与预处理校验

1. **提取数据集**：通过 `datasets/data/` 目录结构，验证是否存在 `train`, `val`, `test` 三个核心图像及标签缓存资料。
2. **配置文件纠偏**：此步骤被自动化。引擎执行时遇到基于文件系统迁移动态产生错乱绝对路径等痛点，其内的 `train.py` 中的探针将强制挂钩覆写 `datasets/data/data.yaml` 中的文件寻址索引为最合规的当前系统映射。

## 步骤二：环境底座拉起 
对于拥有前沿图形显卡（Blackwell 50系列等）计算设备的组员，我们已在 `docs/RTX5060_环境配置指南.md` 存放了防范 PyTorch 断代的最佳实践：
1. 确保使用 `uv` 环境包管理。
2. 运行 `uv sync --no-install-package torch --no-install-package torchvision` 加载非推理支持组件库。
3. 运行专用源安装支持新架构核心的运算内核（`torch>=2.7.0` 及配套 `cu128` 硬件管线）。

## 步骤三：启动显卡满载微调 (Fine-Tuning)
1. 在项目根目录（`d:\SITP\SITPproject\`）直接执行命令行：
   ```powershell
   uv run --no-sync pavement-train
   ```
2. 系统将触发硬件嗅探接管（输出 `✅ 成功接管本地算力硬件: ...`），启动针对 640 分辨率的高压 GPU 多线程并发训练：
   - 每轮 Epoch 生成详细计算耗时估算、各项 Loss（定位/分割/分类）的数据下降图。
   - 训练期间的权重的临时检查点会自动记录。
3. 等待跑完 100 轮（以获得我们模型目前最高的 71.3% 的最佳重构精度）。

## 步骤四：一键抽调“学术化实验图解”
默认 YOLO 导出图表分辨率较糙，无法满足 SITP 的结题要求。
1. 在终端直接输入我们特制的自动化绘图工具指令：
   ```powershell
   uv run pavement-plot
   ```
2. 引擎将会潜入 `runs/segment/train(对应编号)/results.csv` 读取训练日志的底层 Pandas 指标矩阵。
3. 通过自制 `Seaborn` 数据降噪转换程序处理后，它会在该目录下生成一套符合 IEEE 论文排版标准、已规避本地化乱码与高对比色盲保护的多边形性能指标高 DPI 及无损矢量级PDF评估报表。

## 步骤五：完成权重实装测试
1. 进入 `runs/segment/train(对应编号)/weights/`。提取你辛勤炼丹产出的 `best.pt` 复制至项目的 `weights/` 目录；为了与基座隔离，可更名为 `my_train_best.pt`。
2. 启动系统交互前端：
   ```powershell
   uv run --no-sync pavement-web
   ```
3. 在左手 **“模型选择 / 自定义权重(.pt)”** 大面板中呼出侧栏，拖拽传入刚刚我们导出的自定义文件。此时底层自动刷新显存状态并不再读取低版本模型；选择任一输入模式即可在界面与 DeepSeek 分析引擎上体验并对比评测自主模型的检测效果。
