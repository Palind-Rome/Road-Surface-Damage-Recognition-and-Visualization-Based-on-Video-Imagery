# RTX 5060 (Blackwell / sm_120) 深度学习环境配置踩坑实录

> **最后更新：2026年3月6日**  
> **硬件：NVIDIA GeForce RTX 5060 Laptop GPU (8GB VRAM, CUDA Capability sm_120)**  
> **系统：Windows 11 / Python 3.10 / uv 包管理器**

---

## ⚠️ 核心问题

RTX 5060 属于 NVIDIA **Blackwell 架构**，计算能力代号为 **sm_120**。  
这是截至 2026 年初**最新一代的 GPU 架构**，导致绝大多数常规安装的 PyTorch 版本无法在该显卡上运行。

### 症状表现

运行任何将张量移至 GPU 的操作（如 `model.to('cuda')`）时，报错：
```
RuntimeError: CUDA error: no kernel image is available for execution on the device
```

同时伴随以下警告：
```
NVIDIA GeForce RTX 5060 Laptop GPU with CUDA capability sm_120 is not compatible 
with the current PyTorch installation.
The current PyTorch install supports CUDA capabilities sm_50 ... sm_90.
Please install PyTorch with a following CUDA configurations: 12.8 13.0
```

### 根因分析

| CUDA 版本 | 支持的最高架构 | 能否驱动 5060 (sm_120) |
|-----------|---------------|----------------------|
| cu121     | sm_90         | ❌ 不行              |
| cu124     | sm_90         | ❌ 不行              |
| cu126     | sm_90         | ❌ 不行              |
| **cu128** | **sm_120**    | **✅ 可以！**        |

**结论：RTX 5060 必须使用 CUDA 12.8 (cu128) 编译的 PyTorch 轮子。**

## ✅ 最终正确的配置步骤

### 第一步：确认 pyproject.toml 配置

```toml
requires-python = ">=3.10,<3.13"

dependencies = [
    # ...其他依赖...
    "torch>=2.0.0",
    "torchvision>=0.15.0",
    "timm==0.9.16",          # 必须锁死版本！>=1.0 会导致 uv resolver 卡死
    # ...
]

[tool.uv.sources]
torch = { index = "pytorch" }
torchvision = { index = "pytorch" }

[tool.uv]
prerelease = "allow"                          # 允许安装预发布版本
environments = ["sys_platform == 'win32'"]    # 阻止 uv 跨平台解析 (避免 triton/linux 报错)

[[tool.uv.index]]
name = "pytorch"
url = "https://download.pytorch.org/whl/cu128"   # ← 关键！必须是 cu128！
explicit = true
```

### 第二步：安装 PyTorch (cu128)

```powershell
# 进入项目目录
cd D:\SITP\SITPproject

# 强制安装 cu128 版本的 PyTorch（必须加 --reinstall，否则 uv 会跳过已有版本）
uv pip install "torch>=2.7.0" "torchvision>=0.22.0" --reinstall --index-url https://download.pytorch.org/whl/cu128
```

> **⚠️ 重要提示：**
> - 这个包大约 2-3 GB，请耐心等待下载完成。
> - 必须加 `--reinstall`！否则如果本地已有其他 CUDA 版本的 torch，uv 会直接跳过不下载。
> - 安装完成后，确认输出中包含 `+cu128` 字样。

### 第三步：验证安装

```powershell
uv run --no-sync python -c "import torch; print(torch.__version__); print('CUDA:', torch.cuda.is_available()); print('GPU:', torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'N/A')"
```

**预期输出：**
```
2.x.x+cu128
CUDA: True
GPU: NVIDIA GeForce RTX 5060 Laptop GPU
```

### 第四步：运行项目

```powershell
# 必须加 --no-sync！否则 uv 会尝试重新解析依赖并可能覆盖掉 cu128 的 torch
uv run --no-sync pavement-train
uv run --no-sync pavement-web
```

## 🚨 踩过的坑（血泪教训）

### 坑 1：uv sync 会自动覆盖手动安装的 torch

**现象：** 用 `uv pip install` 装好了 cu128 版 torch，但一执行 `uv run pavement-train`，uv 又把它卸了换成 cu126 或 cpu 版。

**原因：** `uv run` 默认会先执行 `uv sync`，重新按 lockfile 解析依赖。如果 lockfile 里锁的是旧版本，它会"好心"帮你替换回去。

**解决：** 运行时始终加上 `--no-sync`：
```powershell
uv run --no-sync pavement-train
```

### 坑 2：uv sync 报 triton/linux 解析失败

**现象：**
```
No solution found when resolving dependencies for split 
(markers: python_full_version >= '3.14' and sys_platform != 'win32'):
Because there is no version of triton{sys_platform == 'linux'}==3.6.0+git9844da95...
```

**原因：** uv 的依赖解析器会模拟所有平台（包括 Linux），而 PyTorch nightly 在 Linux 下依赖一个叫 `triton` 的包，该包版本号不合规导致解析器崩溃。

**解决：** 在 `pyproject.toml` 中限定环境：
```toml
[tool.uv]
environments = ["sys_platform == 'win32'"]
```

### 坑 3：timm 版本过新导致 uv resolver 卡死

**现象：** `uv sync` 或 `uv run` 卡在 `⠸ timm==1.0.25` 不动，一直转圈。

**原因：** `timm>=0.9.0` 允许 uv 去拉取 1.0.25 的最新版，该版本的依赖图与 PyTorch nightly/cu128 存在冲突，导致解析器陷入死循环。

**解决：** 锁死 timm 版本：
```toml
"timm==0.9.16",
```

### 坑 4：cu126 显示 CUDA True 但实际不能用

**现象：** `torch.cuda.is_available()` 返回 `True`，`torch.cuda.get_device_name(0)` 也正常显示 RTX 5060，但一旦执行 `model.to('cuda')` 就报错 `no kernel image`。

**原因：** cu126 的 PyTorch 确实能检测到你的 GPU 硬件（通过 NVIDIA 驱动），但它没有编译 sm_120 的 CUDA 计算内核，所以"看得见用不了"。

**解决：** 只有 cu128 的轮子才编译了 sm_120 内核。

### 坑 5：`uv pip install` 不加 --reinstall 不会更换 CUDA 版本

**现象：** 执行 `uv pip install torch --index-url .../cu128` 后输出 `Audited 2 packages in 12ms`，版本还是 cu126。

**原因：** 本地已有满足版本约束的 torch（比如 2.10.0+cu126 满足 `>=2.7.0`），uv 认为不需要重新下载。

**解决：** 必须加 `--reinstall` 强制重装：
```powershell
uv pip install "torch>=2.7.0" --reinstall --index-url https://download.pytorch.org/whl/cu128
```

## 📋 快速恢复清单（TL;DR）

如果你换了电脑或重建了环境，按以下顺序执行即可：

```powershell
# 1. 进入项目目录
cd D:\SITP\SITPproject

# 2. 同步基础依赖（不含 torch）
uv sync --no-install-package torch --no-install-package torchvision

# 3. 安装 cu128 版 PyTorch（约 2-3 GB）
uv pip install "torch>=2.7.0" "torchvision>=0.22.0" --reinstall --index-url https://download.pytorch.org/whl/cu128

# 4. 锁定 timm 版本
uv pip install timm==0.9.16

# 5. 验证
uv run --no-sync python -c "import torch; print(torch.__version__, torch.cuda.is_available())"

# 6. 开跑
uv run --no-sync pavement-train
uv run --no-sync pavement-web
```
