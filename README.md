# 同济大学SITP项目：基于视频图像的路面病害识别与展示系统

## 📌 项目简介
本项目由**同济大学SITP项目团队**自主设计与研发，旨在通过前沿的计算机视觉技术（YOLOv8 实例分割）与自然语言大模型（DeepSeek-V3），打造一套高精度、高并发、易用的路面病害智能评估与报告系统。系统支持对多源多模态输入（图片、视频、实时摄像头流）进行毫秒级推理解析，并能够生成面向工程管理的专业级分析报告。

## 🌟 核心架构优势与技术亮点
本项目针对大规模工业级检测场景进行了深度的底层重写与优化，具备极其卓越的性能与稳定性表现：

1. **🚀 极速模型驻留与状态机架构 (Lightning-fast Model Caching)**
   - 自主研发了基于前端状态引擎的底层接管方案，彻底解决了传统 Web 框架在组件交互时频繁重载几十MB深度学习大模型导致的内存泄漏与显存抖动（VRAM Thrashing）问题。
   - 实现 UI 控件拖拽、检测模式实时切换的「零延迟」响应，整体推理与交互引达到了极高的丝滑度。

2. **🛡️ 深度学习安全防御穿透机制 (Deep Learning Safe-Load Protocol)**
   - 针对 PyTorch 2.6+ 引入的严苛安全反序列化拦截（WeightsUnpickler 限制），项目在底层实现了自定义的安全权重读取协议。
   - 完美兼容与绕过核心网络模块中的全局加载限制，大幅领跑同类开源架构方案，确保系统在最新 Python 运行时下的极高健壮性。

3. **📊 现代化动态数据流控视图 (Modern Dynamic DataGrid)**
   - 摒弃了会撑爆网页渲染树（DOM）的传统静态 HTML 表格渲染体系，全新研发了无界限滚动 DataFrame 控制层。
   - 利用动态虚拟化列表技术，即使在复杂路况下生成数千条病害检测记录，前端依然保持高帧率渲染，并支持动态吸顶表头与平滑滚动。

4. **✨ 高性能交互界面引擎 (High-Performance UI Engine)**
   - 进行了深度 UI 性能优化与前端代码重构，清除了导致原生组件互斥的 CSS 污染。
   - 彻底修复并优化了侧边栏渲染刷新、单选按钮（Radio）状态丢失与锁死的交互 BUG，为用户提供纯净、无感刷新的桌面级应用沉浸体验。

## 🛠️ 环境依赖与项目运行

本项目采用了先进的现代化 Python 依赖管理工具 `uv` 构建，实现了极速、完全隔离的标准化运行环境。

### 1. 准备运行环境
确保系统已全局安装 `uv`（极速包管理器）：
```powershell
# Windows PowerShell 下一键安装
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

### 2. 核心资源目录说明
在运行前，请确保项目根目录下存在以下核心资产（模型与数据集）资源：
- `weights/`：存放 YOLO 预训练模型（如 `yolov8s-seg.pt` 及 `yolov8s.pt` 等）。
- `datasets/`：存放训练与测试数据集及其 YAML 配置文件。
- `ultralytics/`：底层网络构架源码支撑库。
- `icon/`：系统 UI 静态素材与底图。

### 3. 一键启动系统 (Web UI)
借助项目根目录下的 `pyproject.toml` 标准化配置，无需繁琐的虚拟环境激活指令，只需一行命令即可全自动解析依赖并拉起系统前端：

```powershell
uv run pavement-web
```

> 🎯 **运行效果**：首次执行将自动下载环境与算力依赖（支持按需匹配对应系统的 ONNXRuntime）。命令执行完成后，浏览器将自动打开 `http://localhost:8501/`，即可体验完整的智能检测工作流。

### 4. 启动独立重训引擎（自定义微调与训练）
为了适配不同拍摄环境与特定路面材料，本项目内置了**全自动路径修正训练引擎**。如果需要通过新增的路面病害数据集对检测模型进行 Fine-tuning（微调）：

1. **准备数据**：请确保你的数据集按 YOLO 格式放置于 `datasets/data/` 目录下（包含 `train`, `val` 以及对应的 `data.yaml`）。
2. **调整参数**：打开 `src/pavement_detect/train.py`，根据你的显卡显存调整 `epochs`（训练轮数，推荐 50-100）和 `batch`（批次大小，如 8 或 16）。
3. **启动训练**：
   ```powershell
   uv run pavement-train
   ```
   *(引擎具备“路径漂移自动纠正”技术，会自动修复 `data.yaml` 中由于移动设备或拷贝带来的绝对路径报错，确保 GPU 训练稳定进行)*
4. **提取权重与图表**：训练结束后，最新的最优检测权重 `best.pt` 将被保存在 `runs/segment/train` 目录中。

### 5. 生成高质量学术图表
默认生成的验证图表分辨率较低、无中文字体且缺乏学术排版。为此，本项目内置了基于 Seaborn 重写的学术级数据清洗与高精绘图脚本，**已自动适配 Windows 黑体/微软雅黑等中文字体渲染（完美解决豆腐块乱码问题）**。
在完成上述模型训练后，在根目录直接运行以下指令：
```powershell
uv run pavement-plot
```
系统将自动读取你上一次训练产生的 `results.csv` 数据，在 `runs/segment/train/academic_plots` 目录下重绘以下图表：
- `Academic_Loss_Curves.png / .pdf`：包含边界定位、多边形分割、类别分类的三维度 Loss 下降对比神图。
- `Academic_Accuracy_mAP.png / .pdf`：验证集模型综合精度 mAP@0.5 与 mAP@0.5:0.95 走势图。
*(支持 600DPI 极高分辨率与 PDF 矢量无损导出格式，双语标注，极其适合直接插入 Word 结项报告与答辩 PPT 中。)*

## 🧠 AI 报告生成引擎 (DeepSeek-V3 赋能)
系统深度融合了目前业界领先的大型语言模型 API，对多边形掩码面积、病害类别、检测置信度等抽象的视觉几何数据进行工程化解读。
- 自动化生成**专业详尽的管理业务分析报表**。
- 支持针对路面病害风险度发起**多轮连续对话与追问**，流式响应渲染。
- API 密钥及远端推理节点配置参数可通过 `src/pavement_detect/config.py` 自定义无缝配置。
